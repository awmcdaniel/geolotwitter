Figuring out OAuth

twitter uses OAuth to ensure that programs are making authorized calls to its API. There’s a (large) set of documentation on twitter’s developer site devoted to authentication. As you’ll see if you visit, there’s some pretty complex stuff going on with encoding, access tokens, and more.

Luckily for us, twitter recommends using a client library to handle the authentication, and maintains a list of libraries in different languages. We’re going to use the twitteroauth library by Abraham Williams. Following the documentation available on Abraham’s TwitterOAuth site, we download the latest release (0.5.1 as of the writing of this guide) and create a new directory on our server called twitteroauth. We then unpack the latest release to that directory.

We’ll need to include two lines at the start of our PHP code to access the library: 

require "twitteroauth/autoload.php";

use Abraham\TwitterOAuth\TwitterOAuth;

In the above example, twitteroauth/autoload.php should be replaced with the relative path to the autoload.php file on your server.

Now we can get on with our request!
